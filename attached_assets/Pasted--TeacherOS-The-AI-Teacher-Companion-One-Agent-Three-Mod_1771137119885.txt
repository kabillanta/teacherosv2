## ğŸ“š NCF Context Injection

> *Not a vector DB. Not traditional RAG. Something smarter for this use case.*

We pre-process NCF 2023 (~600 pages) into **topic-tagged text chunks** and inject relevant sections into the agent's context window using Gemini's **1M token capacity**.

| Component | Detail |
|-----------|--------|
| **Source** | NCF 2023 PDF, manually chunked by topic |
| **Storage** | `backend/ncf_data/` â€” plain `.txt` files |
| **Retrieval** | Keyword/tag matching per request type |
| **Injection** | Loaded into Gemini's context at inference time |

**Why not full RAG?** For a 600-page document with clear topic boundaries, keyword matching + large context windows **outperforms** complex embedding pipelines. No vector DB, no embedding model, no infrastructure overhead. This is an intentional architectural decision â€” simpler is faster for hackathons AND for production.

**Files:**

| File | Content |
|------|---------|
| `pedagogy_approaches.txt` | Teaching methodologies recommended by NCF |
| `assessment_practices.txt` | Formative & summative assessment guidelines |
| `classroom_management.txt` | NCF's classroom environment recommendations |
| `stage_wise_guidelines.txt` | Foundational / Preparatory / Middle / Secondary |
| `competency_framework.txt` | Learning outcomes & competencies by subject |
